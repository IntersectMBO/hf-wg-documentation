#### Key materials

* Summary
  * Ogmios and Kupo have two paths forward (both providing sufficient support to the community to progress with van Rossem hard fork readiness
    * Tools author is expected to review, adjust and make van Rossem compatible versions based on the original codebase in the week of 22nd June
    * Until that happens, forked versions of those tools (Ogmios [v6.14.0.2](https://github.com/IntersectMBO/ogmios/releases/tag/v6.14.0.2) and Kupo [v2.11.0.1](https://github.com/IntersectMBO/kupo/releases/tag/v2.11.0.1)) can be used to test for van Rossem compatibility
    * It was pointed out that for the time being, forked, van Rossem compatible repos, are going to continue to be considered as viable with IO backed maintenance
  * Issue being reported by Indigo labs, [CPU overspend issue on preprod](https://github.com/IntersectMBO/cardano-node/issues/6600), was close with user confirmed they were using an older lucid with hard coded values which is unrelated to the node
  * Adoption of Node 11.0.1 and DBSync 13.7.1.0 continues to increasy steadily
    * According to [PoolTool](https://pooltool.io/networkhealth), nodes self reporting v11 are at 55% and blocks protocol version 11 in the current epoch is at 76%
    * According to [Cexplorer](https://cexplorer.io/hardfork?block=5_days), block production v11 in the last 5 days is at more than 85%
  * [Readiness tracker](https://docs.google.com/spreadsheets/d/1ECoYFCtrRtFX3BvqfhaZi6FxlXQMoTGP/edit?gid=960883614#gid=960883614) has been updated:
    * Mainnet submission readiness checklist in the summary tab is having its readiness numbers that are based on items considered to be on the critical path
      * Exchanges readiness is represented as sum of liquidity stakes from each exchange that is considered ready
    * Few light wallets and few tools have become ready
    * Dapps continued and focused on adjustments and testing aginst Preview ahead of the PreProd fork where they will continue their testing efforts
  * PreProd hard fork initiation governance action was enacted on **10th June**
  * Go/No-go Mainnet hard fork initiation governance action submission call date - **15th June**
    * Readiness tracker mainnet submission checklist will be adjusted if needed, to be compliant with the hard fork submission policy
